1. Overview

ME17Suite is a professional desktop tool for ECU tuning of the Bosch ME17.8.5 electronic control unit used in Sea-Doo personal watercraft with Rotax ACE engines.

2. Loading Files

Opening a Binary File

  1. Click File ▾ in the toolbar, then Open File 1 (primary binary)
  2. Navigate to your .bin ECU dump file (e.g., dumps/2021/1630ace/300.bin)
  3. ME17Suite validates the file: checks size (1,540,096 bytes), MCU string, SW ID
  4. If valid, the SW ID and description appear in the badge bar
  5. The map library (left panel) populates with all detected maps for your SW variant

File Comparison (ORI vs Tune)

  1. Open the primary file (ORI) as File 1
  2. Click File ▾ → Open File 2 to load a second file (e.g., NPRo STG2)
  3. Map cells that differ between File 1 and File 2 are highlighted in the heatmap
  4. Use Compare button to see a summary of all changed regions
Note: Both files must have the same file size (1,540,096 bytes) and ideally the same SW layout. Comparing files with different SW IDs (e.g., 10SW066726 vs 10SW040039) is valid but be aware that NPRo also changes firmware code — not just map values.

SW ID10-byte ASCII at 0x001A (e.g., 10SW066726)

3. Map Editor Tab

The Map Editor tab (blue) is the primary interface for viewing and editing calibration maps.

Map Library Panel (left sidebar)

  • Tree view organized by category: Injection, Ignition, Lambda, Torque, RPM Limiter, Fuel, Misc
  • Search box at top — type to filter maps by name
  • Click any map name to load it into the main table
  • Maps with differences (when File 2 is loaded) are marked

Map Table (center)

  • Displays a 2D table (rows = Y axis / load, columns = X axis / RPM)
  • Color-coded heatmap: blue = low values, green = medium, orange/red = high values
  • Axis labels shown along edges (RPM values on top, load % on left)
  • When File 2 is loaded: changed cells are highlighted with a different border

Properties Panel (right sidebar)

Three tabs showing information about the selected map:

  • Tab 1 — Map: Name, address, dimensions, format (u8/u16 BE/LE), scale, unit, mirror offset, full description and notes
  • Tab 2 — Hex: Raw hex view of the map bytes in the binary
  • Tab 3 — DTC: DTC-specific information (when DTC Off tab is active)

4. Editing Map Values

Direct Cell Editing

  1. Double-click any cell in the map table
  2. Type the new value (in display units — degrees, %, lambda, etc.)
  3. Press Enter to confirm, or Escape to cancel
  4. The cell turns yellow/highlighted to indicate a pending change
  5. Mirror is automatically updated if the map has a mirror offset

Value Ranges

Each map has defined min/max limits. ME17Suite validates your input:

  • Ignition timing: 0–43.5° BTDC (raw 0–58, scale ×0.75°)
  • Lambda: 0.50–2.00 (Q15 format)
  • Torque: 80–150% (approx.)
  • Rev limiter: 4000–13000 RPM range

Multiple Cell Selection

  • Click and drag to select a range of cells
  • Right-click for context menu: Set all selected to value, Scale selected by %

Saving Changes

  1. Click File ▾ → Save (overwrites current file) or Save As…
  2. A dirty flag indicator in the title bar shows unsaved changes
Tip: Always work from a copy of the original dump. Never overwrite your ORI baseline!

5. Undo / Redo

  • Press Ctrl+Z to undo the last change
  • Press Ctrl+Y or Ctrl+Shift+Z to redo
  • Undo history is maintained per session (cleared on new file load)
  • Multiple levels of undo are supported

6. CSV Export

Export any map to CSV for use in Excel, spreadsheet tools, or further analysis:

  1. Select a map in the library and load it
  2. Right-click on the map table → Export to CSV
  3. Choose a file path and save
  4. The CSV includes axis labels (RPM, load%) and cell values in display units

7. DTC Off Function

The DTC Off tab allows you to disable specific diagnostic trouble codes (DTCs) from the ECU firmware.

Important: Changing DTCs only modifies the CODE region. No checksum update is required.

How DTC Off Works

For each DTC, the ECU stores:

  1. An enable byte — controls monitoring (0x06 = active, 0x00 = disabled)
  2. A code storage address (main) — where fault code is stored when triggered
  3. A mirror address = main_addr + 0x0366 — redundant storage

DTC Off writes 0x00 to the enable byte, and clears both main and mirror code addresses.

Disabling a DTC

  1. Switch to the DTC Off tab (orange)
  2. Browse the sidebar — DTCs are grouped by P0xxx (65 codes), P1xxx (35 codes), P2xxx (11 codes)
  3. Click a DTC to view details (code, address, enable byte address) in Properties panel
  4. Click Disable to turn off that DTC
  5. The DTC status in the sidebar changes: green = active monitoring, red = DTC OFF

Disable All DTCs

  1. In the DTC Off tab, click the ▾ Advanced dropdown
  2. Select Disable All DTCs
  3. Confirm the action

8. Checksum Update

When is checksum update needed?

Change typeChecksum update?
Map values (CODE region 0x010000–0x05FFFF)NOT needed
DTC OffNOT needed
BOOT region (SW version, bootloader)YES — required

Verifying checksum

  1. Click Checksum in the toolbar
  2. The dialog shows: stored checksum vs computed checksum, and whether they match
  3. For standard map tuning, both should match (green ✓)

Updating checksum

  1. If BOOT was modified (unusual), click Update Checksum
  2. ME17Suite uses a meet-in-the-middle inverse CRC algorithm to compute the new CS
  3. The new CS is written to address 0x0030
Note: The RSA signature at 0x7E7C–0x7EFF (132 bytes) cannot be recomputed — it requires the Bosch private key. Changing BOOT code that requires a new RSA signature will result in ECU rejection.

Checksum algorithm details

Algorithm:  CRC32-HDLC
Polynomial: 0xEDB88320 (reflected 0x04C11DB7)
Init:       0xFFFFFFFF
XorOut:     0xFFFFFFFF
Region:     BOOT [0x0000, 0x7F00) = 32,512 bytes
Storage:    @ 0x0030 (u32 Big-Endian)
Residue:    0x6E23044F (fixed for all valid ECU files)

9. EEPROM Viewer

The EEPROM viewer reads a 32KB Data Flash (DFlash) dump from the TC1762 MCU. DFlash and PFlash (firmware+maps) are both internal to the same TC1762 chip — they are separate memory banks, not separate chips.

Loading an EEPROM file

  1. Click EEPROM in the toolbar
  2. Load a 32KB EEPROM binary (typically obtained via KTAG/KESS programmer)

Information displayed

  • Hull ID / VIN (e.g., YDV89660E121)
  • ECU serial number (e.g., SF00HM00196)
  • MPEM SW ID and hardware type (064/063/062)
  • First programming date and last update date
  • Programming count
  • Dealer name
  • Operating hours (from circular buffer) in HHH:MM format

Operating hours note

Hours are stored as minutes in a circular buffer. The address varies by hardware type:

062 (GTI older, RXT-X 260)10375091/092xx0x5062 → 0x4562 → 0x1062 (rotacijski, najnoviji prvo)

HW 063 napomena: Spark i GTI SE 155 koriste isti MPEM prefiks (063), ali imaju različitu DTC arhitekturu. ODO za Spark čita se iz dva buffer mjesta (0x4562 i 0x0562) — parser uzima veću vrijednost.

10. Important Warnings

CAL Region (0x060000+) — READ ONLY
Despite appearing to contain calibration data, the CAL region contains TriCore AUTOSAR/ASCET compiled bytecode. Writing to this region will corrupt the ECU firmware. All tuning maps are exclusively in the CODE region (0x010000–0x05FFFF).
Always keep a backup
Always keep an unmodified copy of the original ECU dump. Write your modifications to a new file. Never overwrite the original.
Injector deadtime — DO NOT MODIFY
The injector deadtime table at 0x025900 is a hardware calibration constant specific to the physical injectors. Changing it will cause incorrect fuel injection timing and may damage the engine.
CTS NTC lookup — DO NOT MODIFY
The NTC ADC lookup table at 0x0258AA contains hardware calibration for the coolant temperature sensor. Changing it will cause incorrect temperature readings and may trigger false fault codes or prevent overtemp protection.
SC bypass map — which copy to modify
There are 3 copies of the SC bypass map. Only modify 0x0205A8 (active copy) and 0x029993 (extra copy). Do NOT modify 0x020534 (shadow copy) independently — NPRo's tune keeps it at the original value.
Lambda sensor note
Rotax ACE 1630 and 900 HO engines do NOT have a physical lambda (O2) sensor. The lambda map in the ECU (0x0266F0) is an open-loop AFR target table. Changes directly affect the fuel mixture without any feedback correction.
